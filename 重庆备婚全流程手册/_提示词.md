# 重庆备婚全流程手册 · 写作提示词与执行方案存档

> 本文件为专栏写作过程使用的提示词、执行方案与验收记录存档，不进入专栏展示（下划线前缀文件被 check_book_structure.py 豁免、被 build_site.py 排除出站点）。保留用于复盘与迭代。

## 一、专栏定位

- 主题：重庆忠县男方 + 重庆彭水女方备婚全流程手册
- 读者：相处一年多、双方家长已见面、考虑旅行婚礼或传统婚礼的情侣
- 核心目标：让这对重庆情侣拿到一本「能照着做、不踩风俗雷、不丢面子、能留下深刻印象」的备婚手册——每一篇都能落地，而不是信息搬运
- archetype：modern（现代方法论桶）
- 输出目录：`/workspace/output/重庆备婚全流程手册/`
- 输出：47 篇正文 + 3 个辅助文件（_meta.yaml / _参考资料.md / _提示词.md）

## 二、执行流程

1. **计划评审**：主 Agent 写计划等用户确认
2. **隐藏资产**：建 _meta.yaml / _参考资料.md / _提示词.md
3. **并行写作**：8 个 subagent 分模块并行写 47 篇
4. **章回体灵魂标题评分**：<3 分重写
5. **六维度质检**：真实性 / 可读性 / 顺序 / 引用克制 / 灵魂 / 一致性
6. **修复 + 校验**：check_book_structure --strict
7. **沉淀 loop_log**

## 三、写作硬约束（写入每个写作 subagent 的 query）

1. 目录：`/workspace/output/重庆备婚全流程手册/`
2. 文件命名：两级路径 `{chapter}_{event}.md`，中文，下划线 `_` 分隔，禁止用 `-`，禁止子目录
3. 每篇必须有完整 frontmatter（title/book/chapter/event/sort/chapter_sort/created_at/source_agents 八字段）
4. archetype: modern（现代方法论桶）
5. 正文中文，每篇 2000-3500 字
6. 5 段结构固定段名：入戏 / 破题 / 方法论 / 避坑 / 践行
7. 文末必须有 `## 参考来源` 标题（不是"参考资料""参考文献"）
8. 禁内联跳转（"（见XX篇）""（详见下章）"等带括号跳转提示）；提及他篇用自然语句"（XX 篇展开）"
9. 禁用"底层逻辑""底层操作系统"
10. 术语词序铁律："英文+中文"词序（如"Gottman 法则"而非"戈特曼Gottman 法则"）
11. 子标题"名词：洞察短语"
12. 每篇开头一句边界声明"本篇覆盖 X，不覆盖 Y（XX 篇展开）"
13. 文末加 `> 信息截止：2026-07`

## 四、重庆本地化要点（必须落实）

1. **忠县（男方）**：汉族巴渝风俗，长江沿岸，彩礼中等偏低（区间 1-6 万，常见 3 万左右"万里挑一"），重提亲、订婚、回门
2. **彭水（女方）**：苗族土家族自治县，少数民族风俗保留较多，可能有拦门酒、哭嫁、对歌、苗绣嫁衣等元素；彩礼区间与忠县接近但礼数更繁
3. **两地距离**：忠县 ↔ 彭水约 280 公里，自驾约 4 小时，接亲路线需考虑跨县物流与住宿
4. **面子文化**：重庆人重人情面子，彩礼数字、酒席桌数、接亲车队、敬酒顺序都关乎两家面子，必须给"商量话术"而非死规则
5. **不能写错**：彭水是苗族土家族自治县，不要写成"彭水苗族县"或"彭水土家族县"；不要把苗族银饰与土家族织锦混淆
